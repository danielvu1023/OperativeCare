Severity Palette

Four-tier severity system with background, border, and text tokens per level. Calibrated for WCAG AA contrast in well-lit clinical environments. Each level has 3 tokens: -bg, -border, -text.

Buttons

Four variants. Primary for the single main action per view. Secondary for supporting actions (sort, filter). Ghost for tertiary inline links. Destructive for irreversible actions only.

Vital Cards

The core monitoring unit. Left accent bar signals status at a glance. Trend arrow reflects 6-reading rolling average (2.5% threshold). Sparkline height is 64px for clinical readability. Abnormal cards are elevated with tinted backgrounds and box-shadow.

Components

Status Bar

Full-width contextual band placed directly below PatientHeader. Provides ambient severity awareness — the patient's current state is readable before any detail is read. Critical state uses a solid filled background for maximum urgency.

✓ All Vitals Normal — Patient stable, no intervention required Route → Nurse (Floor)
▲ Warning — Heart rate elevated above 110 bpm threshold Route → Charge Nurse
!! Urgent — Systolic BP declining, respiratory rate elevated Route → Resident Physician
✕ CRITICAL — Systolic BP 78 mmHg, SpO₂ 86% — Rapid Response Required Route → Attending + Rapid Response

Components

Alert Rows

Left-border accent creates consistent visual vocabulary with vital cards. Critical and urgent rows carry a tinted background for additional weight. Sorted by severity tier first, then by recency. Timestamps are relative.

Components

Navigation

The nav is an invariant dark zone — it never adopts the light theme. This creates a persistent visual anchor that separates chrome from content, and maintains orientation regardless of the content area's severity state.

Patient Header

Establishes patient identity at the top of every clinical view. Name uses the largest type size on the page. Allergy is a red badge — visually equal to a warning alert.

Token Reference

All tokens defined in styles/globals.css under :root. Tailwind utilities are registered via @theme inline. Use the --color-* prefix for Tailwind, bare var(--*) for vanilla CSS.

/* styles/globals.css — :root */ /* Surfaces */ --background: #f8fafc; /* bg-background */ --surface: #ffffff; /* bg-surface */ --surface-raised: #f1f5f9; /* bg-surface-raised */ --border: #e2e8f0; /* border-border */ --border-strong: #cbd5e1; /* border-border-strong */ /* Foreground */ --foreground: #0f172a; /* text-foreground */ --foreground-muted: #475569; /* text-foreground-muted */ --foreground-subtle: #94a3b8; /* text-foreground-subtle */ /* Primary */ --primary: #1e40af; /* bg-primary / text-primary */ --primary-hover: #1d4ed8; --primary-light: #eff6ff; /* bg-primary-light */ /* Severity — normal */ --severity-normal-bg: #f0fdf4; --severity-normal-border: #16a34a; --severity-normal-text: #15803d; /* Severity — warning */ --severity-warning-bg: #fffbeb; --severity-warning-border:#d97706; --severity-warning-text: #b45309; /* Severity — urgent */ --severity-urgent-bg: #fff7ed; --severity-urgent-border: #ea580c; --severity-urgent-text: #c2410c; /* Severity — critical */ --severity-critical-bg: #fef2f2; --severity-critical-border:#dc2626; --severity-critical-text: #b91c1c; /* Navigation (invariant dark) */ --nav-bg: #0f172a; --nav-border: #1e293b; --nav-text: #94a3b8; --nav-text-active: #ffffff; --nav-item-active: #1e293b;
